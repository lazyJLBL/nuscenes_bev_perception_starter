# BEV（鸟瞰图）详解

> 本文面向自动驾驶初学者，解释 BEV 的概念和在自动驾驶中的应用。

## 1. BEV 是什么

BEV = **Bird's Eye View**（鸟瞰图），即从正上方向下看的视角。

想象你在高空俯视一个十字路口：
- 你能看到所有车辆的位置和朝向
- 你能看到道路布局和车道线
- 你看不到物体的高度信息（3D 压缩成了 2D）

在自动驾驶中，BEV 是一个非常重要的**中间表示**。

## 2. 为什么自动驾驶喜欢 BEV

### 2.1 统一的空间表示
- 不同传感器（摄像头、LiDAR）的数据可以投影到同一个 BEV 平面
- 方便进行**多传感器融合**

### 2.2 便于规划
- 自动驾驶的规划模块需要在 2D 平面上做路径规划
- BEV 视角天然适合路径规划（就像你在地图上画路线）

### 2.3 没有遮挡问题
- 摄像头图像中，远处的物体可能被近处的物体遮挡
- BEV 视角下，只要 LiDAR 能探测到，物体就不会被遮挡

### 2.4 尺度一致
- 摄像头图像中，远处的物体看起来更小（透视效果）
- BEV 中，物体的大小与实际物理尺寸成正比

## 3. BEV 和图像视角的区别

| 对比项 | 摄像头图像 (Perspective) | BEV (鸟瞰图) |
|--------|--------------------------|---------------|
| 视角 | 从车辆水平方向看 | 从正上方向下看 |
| 尺度 | 远小近大 | 尺度统一 |
| 遮挡 | 有遮挡 | 几乎无遮挡 |
| 深度 | 丢失深度信息 | 保留 XY 位置 |
| 高度 | 可见高度信息 | 高度被压缩 |
| 适合 | 图像分类/检测 | 规划/预测 |

## 4. LiDAR BEV 和 Camera BEV

### LiDAR BEV（本项目使用的方式）
- 直接将 LiDAR 3D 点云投影到 XY 平面
- 简单、快速、准确
- 缺点：LiDAR 昂贵，且远处点稀疏

### Camera BEV
- 从 2D 图像通过深度估计或特征变换生成 BEV
- 代表工作：BEVFormer、BEVDet、BEVFusion
- 优点：成本低（只需摄像头）
- 缺点：需要深度学习模型，精度依赖模型能力

### 融合 BEV
- 同时使用 LiDAR 和 Camera 生成 BEV
- 代表工作：BEVFusion、TransFusion
- 优点：结合两者优势

## 5. BEV 三通道说明

本项目从 LiDAR 点云生成的 BEV 包含三个通道：

### 5.1 高度图 (Height Map) — R 通道

```
每个 BEV 像素 = 该区域内最高点的 Z 坐标
```

- 高处物体（如车顶、建筑物）显示为亮色
- 地面显示为暗色
- 用途：区分地面和障碍物

### 5.2 密度图 (Density Map) — G 通道

```
每个 BEV 像素 = 该区域内的 LiDAR 点数
```

- 点多的区域（如障碍物表面）显示为亮色
- 空旷区域显示为暗色
- 用途：区分有物体和无物体的区域

### 5.3 强度图 (Intensity Map) — B 通道

```
每个 BEV 像素 = 该区域内点的平均反射强度
```

- 高反射率的物体（如车牌、道路标线）显示为亮色
- 低反射率的物体（如沥青路面）显示为暗色
- 用途：区分不同材质的物体

## 6. BEV 参数设置

```yaml
bev:
  x_range: [-50.0, 50.0]   # 前后各 50 米
  y_range: [-50.0, 50.0]   # 左右各 50 米
  z_range: [-5.0, 3.0]     # 地下 5 米到地上 3 米
  resolution: 0.2           # 每个像素 0.2 米
```

- **范围**: 以 ego vehicle 为中心，覆盖 100m × 100m 的区域
- **分辨率**: 0.2m/pixel 表示每个像素代表 20cm × 20cm 的地面区域
- **图像尺寸**: 100m / 0.2m = 500 像素，即生成 500 × 500 的 BEV 图像

## 7. BEV 在自动驾驶流水线中的位置

```
传感器数据 (Camera + LiDAR)
        ↓
   特征提取
        ↓
   BEV 特征图 ← 这就是 BEV 的位置
        ↓
   ┌────┴────┐
   ↓         ↓
  检测      分割
   ↓         ↓
  跟踪     占据网格
   ↓         ↓
  预测      地图
   ↓         ↓
   └────┬────┘
        ↓
      规划
        ↓
      控制
```

BEV 是连接**感知**和**规划**的桥梁。
